Show events

  • MESSAGE_CONTENT does not represent individual events, but rather affects what data is present for events that could contain message content fields.
  • For example, if the MESSAGE_CONTENT intent is enabled, events like MESSAGE_CREATE, MESSAGE_UPDATE, and MESSAGE_DELETE will include the actual content of the message in their data. However, if the MESSAGE_CONTENT intent is not enabled, these events will only provide limited information about the message, such as its ID or timestamp, without revealing the actual content.
